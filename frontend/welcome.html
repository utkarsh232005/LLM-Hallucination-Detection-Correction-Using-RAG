How It Works

From query to verified truth in seconds.

01
🗣️
User Submits Query

Your question is sent via SSE stream to the Flask backend running locally on port 8080. No data leaves your machine for the LLM step.

02
🦙
Llama Generates Answer

llama3.2:1b via Ollama produces an initial response with full local inference — no API key needed for generation.

03
🌐
Web Sources Retrieved

SerpAPI fetches real-time Google results. LangChain's WebBaseLoader scrapes the pages and nomic-embed-text embeds them into Pinecone.

04
🔬
NLI Model Judges

A HuggingFace CrossEncoder evaluates each (context, claim) pair and classifies: Entailment, Neutral, or Contradiction.

05
🔧
RAG Correction

If hallucination is detected, the system automatically runs a RAG pass — grounding the answer in verified web sources and returning a corrected response.

06
🗄️
Logged to MySQL

Every interaction — score, classification, sources, response time — is persisted to MySQL via the Node.js API on port 3001 for full audit trail.

✅ Not Hallucinated Entailment

The LLM's response is supported by web sources. The original answer is returned directly — no correction needed. Score logged to MySQL.

⚠️ Hallucination Detected Contradiction

The claim contradicts retrieved context. RAG kicks in automatically, regenerating the answer from verified sources and flagging the log entry.

Quick Start

Up and running in minutes.

# 1. Clone the repo
$ git clone https://github.com/<your-username>/hallucination-rag.git

# 2. Copy and fill in your API keys
$ cp .env.example .env

# 3. Pull Ollama models
$ ollama pull llama3.2:1b && ollama pull nomic-embed-text

# 4. Start the Flask AI backend
$ python backend/server.py
✓ Running at http://127.0.0.1:8080

# 5. Start the Node.js logging API
$ node api/server.js
✓ Running at http://127.0.0.1:3001
